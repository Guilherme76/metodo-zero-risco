# AGENTS.md

## Projeto

Landing Page de Vendas - Método Zero Risco

Objetivo principal:

Converter visitantes em assinantes pagantes do grupo VIP da Comunidade RFP Sports.

## Stack Obrigatória

* Next.js 15+ (App Router)
* React 19
* TypeScript
* Tailwind CSS 4
* Vercel

Não sugerir mudanças de stack.

## Regras de Desenvolvimento

Antes de implementar qualquer funcionalidade:

1. Analisar a estrutura atual do projeto.
2. Verificar componentes existentes.
3. Verificar tipos existentes.
4. Verificar constantes existentes.
5. Evitar duplicação de código.

## Arquitetura

Priorizar:

* Server Components
* Componentização
* Reutilização
* Tipagem forte
* Separação de responsabilidades

Estrutura preferencial:

src/
├── app/
├── components/
├── lib/
├── hooks/
├── types/
├── constants/
├── data/
├── utils/

## Design

Características desejadas:

* Profissional
* Premium
* Alta conversão
* Confiável
* Moderna

Nunca criar interfaces com aparência genérica.

## Responsividade

Mobile-first obrigatório.

Suporte completo para:

* Mobile
* Tablet
* Desktop

## SEO

Sempre considerar:

* Metadata API
* Open Graph
* Twitter Cards
* Structured Data quando aplicável
* HTML semântico

## Performance

Priorizar:

* Server Components
* next/image
* next/font
* Lazy loading quando necessário

Evitar:

* useEffect desnecessário
* Client Components sem necessidade
* Renderizações redundantes

## Acessibilidade

Sempre implementar:

* Navegação por teclado
* Focus visível
* Contraste adequado
* Aria-labels quando necessário

## Landing Page

A estrutura principal deve conter:

1. Hero
2. Problema
3. Solução
4. Como Funciona
5. Benefícios
6. Depoimentos
7. Oferta
8. FAQ
9. Footer

## Psicologia de Conversão

Aplicar:

* Autoridade
* Prova Social
* Escassez
* Urgência
* Garantia
* Redução de Risco

Sem utilizar promessas irreais ou enganosas.

## Fluxo de Trabalho

Antes de gerar código:

1. Explicar o plano.
2. Informar arquivos que serão alterados.
3. Explicar impactos.

Somente depois implementar.

## Qualidade

Todo código gerado deve ser:

* Produção
* Escalável
* Legível
* Bem tipado
* Compatível com Next.js 15+
